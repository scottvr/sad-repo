# Roadmap v0.3 — confound ledger and scaling arcs (2026-07-14)

Successor to `roadmap_v0.2.md` (kill-criterion and branch strategy there
still stand). Written after the pressure and dims grids ran. Numbers and
verdicts: `CURRENT_STATUS.md`.

Operating mode: rapid achievements without academic gatekeeping — every
arc below is sized to a single GPU session where possible, every arc has a
headline *whichever way it comes out*, and every result gets audited
against the confound ledger before it is called an achievement. The
discipline we keep from science: written predictions before runs, controls
next to treatments, and the standing question **"what else, other than the
thing we hope for, would produce exactly this number?"**

## Seeds policy (settled here so we stop re-deciding)

- 3 seeds: smoke/direction checks only. The dims grid at n=3 had ±0.19
  std exactly in the transition band where its signal lived — enough to
  see the shape, not enough to compare allocations.
- **5 seeds: default for every grid** (matches the pressure grid, so grids
  join cleanly on shared arms).
- 10 seeds: any number that goes in a headline table or doc claim.
- Any cross-arm effect smaller than ~2x the seed std is noise. Always
  report ±std; never compare bare means.

## Confound ledger

Each entry: what could make our results look good without being what we
hope for, and the diagnostic that separates the two. Status marks what is
already covered.

**C1. Ceiling / quantization.** 4 probes per task quantizes accuracy in
0.25 steps and compresses every difference near 1.0. *Status: confirmed
real* — the big family deflated retention 1.00 → 0.69. *Remaining fix:*
report retained loss margin alongside accuracy; evaluate on all trained
templates, not one; ≥8 facts/task becomes the default family for claims.

**C2. Label-marginal bias (the most dangerous one).** The answer space is
tiny (6–12 single-token colors). A degenerate adapter that merely shifts
output logits toward the task's label set — storing zero facts — scores
~1/labels-in-play, which is ~0.25 on the default family. That is exactly
the plateau the failing arms sit at, meaning **accuracy ≈ 0.25 is not
"some knowledge", it is plausibly zero knowledge**, and any gain below
~0.3 should be read as nothing. Worse, part of every *good* score may be
bias. *Diagnostics (new code, small):*
  - **Bias-only null baseline**: fit a per-task logit-bias-only "adapter"
    (no site adapters at all), compose and route it through the same
    pipeline, and print its row in every summary table. Our numbers only
    mean something as the *gap* above this null.
  - **Foil probes**: untrained CVCV nonce words in the trained templates.
    A fact-storing adapter has no answer for them; a bias adapter
    confidently names a color for anything. Report foil-answer rate next
    to retention (extends the existing off-domain leakage metric).

**C3. Routing = trivial domain classification.** Every template literally
contains the domain name (`{d}`), so the controller's task is 3-way
classification of an explicit token — routing accuracy ~1.0 cannot be
read as "context understanding". *Diagnostics:* domain-blind eval
contexts (paraphrases that imply the domain without naming it); scale to
6–10 domains; routing on overlap words where the domain token is the
*only* disambiguator (partially exercised by the conflict arm — note
verdict 13 in `CURRENT_STATUS.md` flags the same shallowness for the
composed state's conflict pass).

**C4. Memorization masquerading as adaptation.** ~~Paraphrase consistency
is 0.0 in every condition ever run; updates memorize surface forms.~~
**Update 2026-07-15: those numbers measured the frozen base (coherence
probe ran after `reset_adapters()` — see CURRENT_STATUS "Coherence probe
bug"). Corrected re-run same day: coefficient updates show ~chance
paraphrase transfer (the memorization story survives, now honestly
measured), BUT full-rank naive_stack LoRA scores .88 consistency in the
same runs — capacity, not scale, may gate transfer. New cheap diagnostic
to run BEFORE the template-diversity sweep or TinyLlama: k/rank sweep
reading the new `final_evals_heldout` accuracy column.** It remains the
kill-criterion (roadmap v0.2) and the primary honest metric.
*New diagnostic before betting everything on scale:* a train-template
diversity sweep (2 → 5 → 10 templates per fact at fixed scale). If
paraphrase transfer emerges with phrasing diversity, it was a data
problem, not a scale problem — a much cheaper answer than TinyLlama.

**C5. Replay = joint training in disguise.** Confirmed by the fraction
sweep (retention ~linear in fraction; only full rehearsal saturates). The
defensible claim is only the *locality* of the repair (~96 dims, cramming
refuted). *Diagnostics/next:* joint-fit-all-tasks upper bound as a
standing baseline row (sequential+replay should match it; if it exceeds
it something is wrong); pseudo-replay (rehearsal text generated by the
composed model) as the escape from O(T) data retention — this is the
single highest-leverage new mechanism on the board.

**C6. Two-point scaling "law".** The tidy ~4-dims/fact ≈ 0.7 / ~8
dims/fact ≈ 1.0 pattern rests on two coincident points at small n.
*Diagnostic:* the facts × dims grid in Arc 1, plus the `big_k16`
prediction arm already queued in `run_dims.sh` (192 dims × 24 facts →
~1.0 predicted; miss = the law is wrong, also informative).

**C7. Task homogeneity.** Every task is the same relation (nonce→color)
in the same sentence shapes; "3 domains" is cosmetic. Interference and
routing may both behave differently when tasks differ in kind.
*Diagnostic:* mixed relation types (word→color, word→number, word→city,
word→profession) with different template shapes per domain.

**C8. Single-model idiosyncrasy.** Everything so far is distilgpt2.
*Diagnostic:* gpt2 runs with zero code changes; it is a free replication.

## Arcs (ordered; each ~1 GPU session unless noted)

**Arc 0 — hygiene (next session).**
Re-run dims: `bash scripts/run_dims.sh` (now 5 seeds + `big_k16`), and
write artifacts to a synced location this time (gdrive). Implement + run
the bias-only null (C2) and the joint-fit upper bound (C5) on the default
and big families. *Pass:* our replay numbers clear the null by a wide
margin and sit at/below joint-fit. *Fail headline (still useful):* "the
composed-state numbers are mostly label bias" — better to know now.

**Arc 1 — de-confound at distilgpt2 (1–2 sessions).**
- Foil probes + all-template eval + loss margins (C1, C2) wired into
  `eval.py`/`metrics.py` and every summary table.
- Facts × dims grid: {12, 24, 48} facts × {48, 96, 192} dims, replay=1,
  5 seeds — fit the dims-per-fact relationship properly (C6).
- Template-diversity sweep for paraphrase consistency (C4).
- Domain-blind routing eval + 6–10 domains with mixed relations (C3, C7).
*Gate to Arc 2:* results still stand above the null with confounds
controlled.

**Arc 2 — gpt2 replication (~1 session).**
Same battery, zero code changes expected (C8). Also re-fit dims-per-fact:
does the constant move with model width/depth? First real scaling data
point for "capacity needed vs model size".

**Arc 3 — TinyLlama, the kill-criterion test.**
Finish the `tinyllama_readiness.md` items (site-suffix/layer-prefix
mapping, tokenizer label audit). Run the headline battery with the richer
vocabulary. **The one question that matters: does paraphrase consistency
leave 0.0?** Yes → the general-adaptation story lives; no → re-scope to
fact-injection / exact-recall personalization / unlearning per roadmap
v0.2, without regret — the confound work above makes either outcome a
clean result.

**Arc 4 — contingent on Arcs 0–3.**
- Pseudo-replay (C5's escape hatch; also the privacy story).
- The LoRA yardstick from roadmap v0.2: ≥90% of rank-1 LoRA quality at
  ~1/1000 the storage, measured on the Arc 3 setup.
- Reversibility buy-back moves to the `reversible-composition` branch.

## Implementation notes (dev machine is syntax-check only)

(`experiments.py` + a `--bias-only` flag on `run_controller.py`); foil
probes and all-template eval (`data.py`, `eval.py`, `metrics.py`);

domain-blind paraphrases + mixed relations (`data.py`); pseudo-replay
(`train.py`). Every experiment keeps the JSON-under-`artifacts/`
convention so `summarize_sweeps.py` labeling keeps working; new arms get
condition tags from config fields as usual.
